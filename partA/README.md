# А хэсэг — API дизайн ба Кэш сан

## А.1 — Муу API шинжилгээ (6+ алдаа)

### Алдаа №1: Нэршил тогтворгүй ба утгагүй (`usr_mgr`, `get_u`, `do_user_op`)
**Зөрчсөн зарчим:** Нэршил  
`usr_mgr` нь `UserManager` байх ёстой. `get_u` нь юу хийдгийг огт илэрхийлэхгүй. Нэр нь кодыг тайлбарлах ёстой.

### Алдаа №2: `db_conn` ба `users_arr` нь `public`
**Зөрчсөн зарчим:** Мэдээллийн далдлалт  
Дотоод хадгалалтын бүтэц (`db_conn`, `users_arr`) гадагш нээлттэй байна. Хэрэглэгч шууд `mgr.users_arr.push(...)` хийх боломжтой болж, дотоод бүтэц эвдэрнэ.

### Алдаа №3: Boolean flag-ийн дүрмийг зөрчсөн (`flag: 0=create, 1=update...`)
**Зөрчсөн зарчим:** Ойлголтын жин (cognitive load)  
Нэг метод дотор 4 өөр үйлдлийг flag тоогоор ялгах нь аймшигтай. `createUser`, `updateUser`, `deleteUser`, `restoreUser` гэж тусдаа метод байх ёстой.

### Алдаа №4: Буцаах утгын тогтворгүй байдал (`'ERR_404'` string)
**Зөрчсөн зарчим:** Нэршил, алдааны зохицуулалт  
`get_u` нь олдохгүй бол `'ERR_404'` гэсэн string буцаана. Энэ нь хэрэглэгчийг `if (result === 'ERR_404')` гэж шалгахад мэхэлнэ. Exception шиддэг байх ёстой.

### Алдаа №5: `id_or_email` — хоёр зорилготой параметр
**Зөрчсөн зарчим:** Ойлголтын жин  
Нэг параметр хоёр өөр утга авах нь дотор нь `if` салаалж, тест бичихэд хүндрэлтэй. `getUserById` ба `getUserByEmail` гэж тусгаарлах хэрэгтэй.

### Алдаа №6: `find(q)` нь `SQLException` шидэнэ гэж коммент бичсэн
**Зөрчсөн зарчим:** Мэдээллийн далдлалт (abstraction leak)  
`SQLException` нь өгөгдлийн сангийн дотоод нарийн ширийн зүйл. Library хэрэглэгч нь өгөгдлийн санг мэдэх шаардлагагүй — library өөрийнхөө exception ашиглах ёстой.

## А.2 — Кэш сангийн дизайны шийдвэрлэлт

**Сонгосон сэдэв:** Кэш сан — LRU, LFU, TTL  
**Интерфейс:** `Cache` (partA/lib/api/CacheBase.js)  
**Factory:** `CacheFactory.create('lru'|'lfu'|'ttl', options)`

### Дизайны шийдвэрүүд
- Concrete классууд (`LRUCache`, `LFUCache`, `TTLCache`) `module.exports`-д гарахгүй — зөвхөн `CacheFactory`-ээр хандана
- LRU: Map-ийн insertion order ашиглан O(1)
- LFU: `freqMap` + `minFreq` хяналтаар O(1)
- TTL: lazy expiration + setInterval цэвэрлэгч
