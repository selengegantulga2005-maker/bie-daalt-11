## Товч тойм

А хэсэгт муу API дизайны алдааг шинжлэн засварлаж, LRU/LFU/TTL cache санг нэг интерфейс, Factory паттернтайгаар хэрэгжүүллээ. Б хэсэгт номын сангийн зээлийн системийг Express.js дээр REST зарчмаар хэрэгжүүлж, RFC 7807 алдааны формат, OpenAPI 3.0 тодорхойлолт, Postman тест цуглуулгатайгаар дуусгалаа.

## Тулгарсан бэрхшээл

Cache сангийн интерфейс болон хэрэгжилтүүд хоорондоо require хийхэд алдаа гарсан тул үндсэн классыг тусдаа CacheBase.js файлд гаргаж шийдсэн
LFU cache дээр хамгийн бага давтамжтай зүйлийг O(1) хугацаанд олохын тулд minFreq хувьсагч нэмж хянасан

## Дүгнэлт

Энэ бие даалтаар API дизайны зарчмуудыг практикаар туршлаа. Муу API-г шинжлэхдээ нэрлэлт, мэдээллийн далдлалт хэрхэн чухал болохыг ойлгосон. Cache сан хийхдээ нэг интерфейстэй гурван өөр хэрэгжилт бичих нь Factory паттерний ач холбогдлыг харуулсан. Б хэсэгт REST API зохион байгуулах, алдааны формат стандартчилах зэрэг бодит ажлын туршлага олж авсан.
